Section 01

System Overview

The AI Accounting Migration System automates the migration of your financial data from Reckon Desktop to Reckon One. It combines deterministic rules (built from the official Reckon mapping specification) with AI inference to map accounts, detect anomalies, and reconcile trial balances — with human oversight at every uncertain step.

1
Upload your Reckon Desktop data
Chart of Accounts, Journal Entries, Trial Balance — as CSV files
2
Rules Engine applies deterministic transforms
27 COA type mappings, 19 tax codes, code cleanup, mandatory accounts — no AI cost
3
AI maps ambiguous accounts
Dual-model LLM inference for accounts the rules alone cannot resolve
4
Human reviews uncertain mappings (HITL)
Low-confidence items surfaced for accountant approval before proceeding
5
Anomaly detection on journal entries
Scans for 10+ Reckon-specific issues: duplicate refs, GST errors, cutoff violations
6
Trial balance reconciliation + final report
CFO-level variance analysis with PASSED / REVIEW / FAILED status and next steps
✓
Key benefit The system gets smarter with every run. Approved mappings are saved to memory — the next migration of a similar client will resolve most accounts automatically, with no AI calls needed.

Section 04

Run Pipeline — Step-by-Step Guide

This is the main page where you upload data and run the full migration pipeline. Click "🚀 Run pipeline" in the sidebar.

Step 1 — Choose upload mode

Two upload modes are available:

ModeWhen to useWhat to upload
Single file (ALL DATA CSV) You have the full Reckon Desktop export in one CSV One CSV file containing all transaction types
Multiple files You have separate files for accounts, journals, TB 3 separate CSV files: Source accounts, Journal entries, Trial balance
ℹ
No file? Use sample data If you do not upload any file, the system will run with built-in sample Reckon Desktop data. This is useful for testing and demonstrations.

Step 2 — Upload your file(s)

A
Single file mode — Upload ALL DATA CSV
Click "Browse files" and select your Reckon Desktop export CSV. The system automatically detects unique accounts, General Journal entries, and calculates trial balance. You will see a preview of accounts, journals, and TB after upload.
B
Multiple file mode — Use the three tabs
Tab 1 — Source accounts: Upload Chart of Accounts CSV (columns: name, type, code)
Tab 2 — Journal entries: Upload journals CSV (columns: ref, date, desc, amount, type)
Tab 3 — Trial balance: Displays default TB; upload custom if available
C
Upload Target accounts (optional)
You may also upload a Reckon One target accounts CSV. If not uploaded, the system uses default Reckon One account structure automatically.

Step 3 — Run the pipeline

Click the "▶ Run pipeline" button. The pipeline runs in three stages:

StageWhat happensTime
Mapping agentMaps all source accounts to Reckon One targets using rules + AI10–30 sec
Anomaly agentScans journal entries for Reckon-specific issues5–15 sec
Reconcile agentCompares source vs migrated trial balance; generates CFO memo5–10 sec

A live progress bar and log console show each step as it completes.

Step 4 — Review the summary

After the pipeline completes, a summary appears showing:

  • Total accounts processed
  • Number APPROVED — ready for import
  • Number REVIEW — need attention
  • Number ERROR — could not be mapped
  • Count of anomalies detected
⚠
HITL alert If the system finds accounts it is not confident about, a warning message will appear: "X accounts need human review → go to HITL Review tab". Review these before proceeding to import.

Section 07

HITL Review — Human Approval

Click "👤 HITL Review" in the sidebar. This page shows accounts the AI was not confident about — requiring a human accountant to review and approve before the migration proceeds.

When does an account go to HITL?

  • AI confidence score is below the HITL threshold (default 70%)
  • Primary and validator models disagree on the target account
  • No matching rule or memory entry exists for this account type

How to review HITL items

1
Read the AI suggestion
Each card shows: source account details, AI-suggested target, confidence score, and the AI's reasoning.
2
Approve or override
If the AI suggestion is correct — click "Approve". If wrong — select the correct target account from the dropdown and click "Override".
3
Decision is saved to memory
Your decision is immediately saved. The next time a similar account is encountered (same name/type), it will be resolved automatically — no human review needed again.
⚠
Important Do not skip HITL review. Accounts left unreviewed will remain in REVIEW status and should not be imported to Reckon One until resolved.

Section 12

Troubleshooting

ProblemLikely causeSolution
Pipeline fails with "API error" Groq API key missing or rate limit hit Wait 30 seconds and retry. Contact support if repeated.
CSV upload shows 0 accounts Wrong column names in your file Check Section 10 for required column names. Rename columns in your CSV if needed.
All accounts go to HITL New account types not seen before Normal for first run. After you review and approve, memory saves them — next run will auto-resolve.
Reconciliation shows FAILED Mandatory accounts (Item Sales, Item Purchase, Rounding) added in Reckon One but not in source TB Expected behaviour. The 3 mandatory accounts add a small variance — the CFO memo will explain this.
Anomaly: "Invoice_over_75_lines" Invoice has more than 75 line items Reckon One limit is 75 lines. Use Entity Transform → Invoice to auto-consolidate to 1 line.
Anomaly: "Blank_reference" Journal entry has no reference number The system will use the transaction ID (txn_id) as the reference — no action needed unless you prefer custom refs.
App loads but shows blank page Browser cache issue Hard refresh: Ctrl + Shift + R (Windows) or Cmd + Shift + R (Mac)
ℹ
Support For any issues not covered here, please contact your implementation team with a screenshot of the error and the Run ID shown in the pipeline summary.
